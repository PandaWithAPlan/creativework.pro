## 1. Контекст проекта

Creative Work Pro Hub / creativework.pro — статический сайт на Apache (HTML5 + CSS3 + Vanilla JS + лёгкий PHP для форм), который должен реализовать трёхуровневую гибридную модель (Community Hub, Services & Tools, Innovation Lab) и служить входной точкой в экосистему.

Основные источники требований:
- `AGENT.md` — роль агента и общие принципы работы.
- Technical Specification (TZ) — архитектура, структура страниц, технические требования.
- Content Documentation — полный контент для всех страниц.
- Deploy & GitHub Guide — структура репозитория, деплой на Apache.
- Starter Code — базовые шаблоны HTML/CSS/JS/PHP.

### 2025-12-01 — Фаза 2, шаг 2.1: структурный каркас главной страницы
- Создана новая ветка `feature/phase2-home-layout` от `feature/phase1-project-structure` для разработки каркаса главной страницы.
- В `src/index.html` вместо одного placeholder‑блока hero добавлена полная структура секций в соответствии с TZ и Content PDF (пока без детального контента):
  1. `#hero` — основной hero-блок с H1 и placeholder-текстом о том, что здесь будут CTA, метрики и фон.
  2. `#pillars` — секция "Three Pillars" с заголовком и контейнером `.pillars-grid` для трёх карточек (Community Hub, Services & Tools, Innovation Lab).
  3. `#about-preview` — краткий блок "О нас" с заголовком и описанием будущих метрик/логотипов партнёров.
  4. `#verticals` — секция направлений/вертикалей с `.verticals-grid` под 8 карточек дисциплин.
  5. `#benefits` — секция преимуществ с `.benefits-grid` под 4–5 карточек USP.
  6. `#pricing` — секция тарифов с контейнерами `.pricing-toggle` и `.pricing-grid` под переключатель Monthly/Yearly и 3 тарифные карточки.
  7. `#case-studies` — секция кейсов с `.case-studies-grid` (в будущем — слайдер/карточки Марии, Роберта, Сары & Тома).
  8. `#blog-preview` — превью блога с `.blog-preview-grid` для трёх последних статей.
  9. `#home-contact` — финальный блок с формой подписки и краткой контактной информацией.
- Для всех секций добавлены заголовки H2 и короткие placeholder‑описания, указывающие, какой контент сюда будет перенесён из Content PDF.
- В навигации главной страницы активирован пункт "Главная" (`class="is-active"`) и ссылки на секции/страницы приведены к единообразному виду (использование `#hero` для якоря).
- Каркас главной страницы теперь отражает полную воронку: от первого касания (hero) до тарифов, кейсов, блога и финальных CTA, что создаёт готовый скелет для последующей Фазы 3 (перенос реального контента и детальная вёрстка блоков).
