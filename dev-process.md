## 1. Контекст проекта

Creative Work Pro Hub / creativework.pro — статический сайт на Apache (HTML5 + CSS3 + Vanilla JS + лёгкий PHP для форм), который реализует трёхуровневую гибридную модель (Community Hub, Services & Tools, Innovation Lab) и служит входной точкой в экосистему.

Фазы 0–3 закрывают анализ, структуру, каркас страниц и основной контент. Фаза 4 посвящена интерактивности, микро-UX и доведению ключевых сценариев до комфортного для пользователя состояния (без глубокого backend).

### 2025-12-01 — Фаза 4, шаг 4.6: FAQ и юридические страницы
- Для FAQ-страницы (`faq.html`) усилен UX на базе `<details>` и `<summary>`:
  - В `styles.css` добавлен блок стилей для `.faq-section`, `.faq-accordion`, `.faq-item` и `summary`: видимые разделительные линии, аккуратные отступы и курсор pointer.
  - Стандартный маркер `<summary>` скрыт, вместо него используется псевдоэлемент `summary::after` со стрелкой, которая плавно поворачивается при раскрытии.
  - Для `summary` настроены состояния hover и `:focus-visible` (через общие focus-стили для ссылок/кнопок/summary), чтобы клавиатурная навигация и наведение мышью были предсказуемыми.
- Для юридических страниц (`privacy.html` и `terms.html`) введён единый класс `.legal` для основного контента:
  - Ограничена максимальная ширина текста (`max-width: 760px; margin: 0 auto;`), чтобы длинные параграфы было проще читать на широких экранах.
  - Уточнены отступы между заголовками и параграфами (`h2` с дополнительным `margin-top`, усиленный отступ между разделами).
  - Единый стиль списков и параграфов в блоке `.legal`, при этом базовые списки глобально получили разумный `padding-left` и `margin-bottom`.
- Для доступности добавлены глобальные `:focus-visible`-стили для ссылок, кнопок и `summary` в `styles.css` (контрастный outline на 2px с небольшим offset), чтобы все интерактивные элементы были явно видны при навигации с клавиатуры.
