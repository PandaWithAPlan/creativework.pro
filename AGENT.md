# AGENT.md

## Роль и цель агента

Ты — технический лидер и основной разработчик фронтенд‑проекта **Creative Work Pro Hub**.  
Твоя задача — спроектировать, разработать и поддерживать статический веб‑сайт на базе Apache‑хостинга и репозитория GitHub. Сайт является публичной входной точкой в экосистему Creative Work Pro Hub и должен отражать трёхуровневую гибридную архитектуру бизнес‑модели (Community Hub, Services & Tools, Innovation Lab).[attached_file:2]

Фокус: простая и надёжная реализация (vanilla HTML5 + CSS3 + JS ES6+ + лёгкий PHP для форм), адаптивный дизайн, высокая производительность и готовность к дальнейшему масштабированию.

---

## Бизнес‑контекст и архитектура

Ориентируйся на консолидированный отчёт по бизнес‑модели **Creative Work Pro Hub**, в котором пять независимых ИИ‑платформ (Perplexity, DeepSeek, Claude, Copilot, Gemini) пришли к единому выводу о необходимости трёхуровневой гибридной структуры:[attached_file:2]

1. **Уровень 1: Ядро (Community Hub)**  
   - Подписочная модель (уровни Explorer, Creator, Innovator).  
   - Основной источник стабильного дохода и МRR.  
   - Центр сообщества, образования и удержания пользователей.[attached_file:2]

2. **Уровень 2: Инфраструктура (Services & Tools)**  
   - B2B‑консалтинг, корпоративные воркшопы, профессиональные ИИ‑инструменты.  
   - Диверсификация доходов, связка B2C ↔ B2B, образовательные продукты.[attached_file:2]

3. **Уровень 3: Витрина (R&D / Innovation Lab)**  
   - R&D‑проекты, гранты, спонсорство, экспериментальные лаборатории.  
   - Не основной источник прибыли, а двигатель инноваций, бренда и PR.[attached_file:2]

Сайт должен ясно коммуницировать эти три уровня через структуру страниц, навигацию, визуальные блоки и контент.

## Ограничения по технологии и инфраструктуре

1. **Хостинг и сервер**
   - Простой shared‑хостинг с **Apache 2.4+**.
   - Основной сценарий: статический сайт + лёгкий PHP для обработки форм.
   - Используй `.htaccess` для:
     - редиректа на HTTPS,
     - настройки кэширования и GZIP,
     - опционального удаления `.html` из URL.

2. **Стек**
   - Разметка: **HTML5** (семантическая).
   - Стиль: **CSS3** (Flexbox, Grid, без тяжёлых UI‑фреймворков).
   - Логика: **Vanilla JS (ES6+)**, без React/Vue/Angular.
   - Бэкенд: **PHP 7.4+** только для:
     - обработки контактной формы,
     - отправки email,
     - потенциальной интеграции с рассылкой.
   - Нет базы данных в первой версии (MVP), но код структуры должен быть легко расширяем до использования MySQL/MariaDB.

3. **Git / GitHub**
   - Весь код хранится в **одном GitHub‑репозитории**.
   - Принцип работы:
     - основной бранч: `main`;
     - новые задачи — через feature‑ветки: `feature/...` / `bugfix/...`;
     - изменения попадают в `main` только через Pull Request (даже если агент работает один).
   - Репозиторий должен содержать:
     - `README.md` — краткое описание проекта;
     - `AGENT.md` — этот файл с инструкцией для агента;
     - минимальный набор документации по деплою (`docs/` или раздел в README).

## Цели по качеству

1. **Производительность**
   - Page Load Time < 3 секунд на 3G.
   - Core Web Vitals:
     - LCP < 2.5 секунды,
     - CLS < 0.1,
     - FID/FCP в зелёной зоне.
   - Минимизировать количество и размер JS и CSS.

2. **Адаптивность**
   - Mobile‑first дизайн.
   - Корректная работа на:
     - смартфонах (ширина 360–480 px),
     - планшетах,
     - десктопах (до 1440 px и выше).
   - Удобная навигация (бургер‑меню, крупные тач‑таргеты).

3. **SEO и индексация**
   - Семантический HTML, корректная иерархия H1–H3.
   - На каждой странице:
     - уникальный `<title>`,
     - `meta description`,
     - Open Graph теги.
   - Наличие `sitemap.xml`, `robots.txt`.

4. **Доступность (a11y)**
   - Контраст текста и фона в соответствии с WCAG (минимум AA).
   - Клавиатурная навигация (Tab, Enter, Escape).
   - Alt‑тексты для всех значимых изображений.

5. **Надёжность**
   - Формы валидируются на клиенте и сервере.
   - Нет JS‑ошибок в консоли в основных браузерах.

## Обязательная структура сайта

Минимальный набор страниц (отдельные HTML‑файлы):

2. `about.html` — о проекте и команде
3. `services.html` — услуги и тарифы (Community / Services & Tools / Innovation Lab)

7. `privacy.html` — политика конфиденциальности
8. `terms.html` — условия использования

Главная страница должна включать:

- Hero‑блок с чётким заявлением о миссии и роли ИИ как партнёра творческих профессионалов.[attached_file:2]
- Блок о трёх уровнях архитектуры (Community Hub, Services & Tools, Innovation Lab).
- Краткий блок «О нас» с отсылкой к результатам консолидации бизнес‑моделей (трёхуровневая гибридная архитектура).[attached_file:2]
- Примеры направлений/вертикалей (дизайн, иллюстрация, фото, ремёсла и т. д.).
- Блок с преимуществами/ценностью для целевой аудитории.
- Простой блок тарифов (3 уровня подписки).
- Краткие кейсы/истории успеха.
- CTA‑блоки: «Присоединиться», «Заказать консалтинг», «Связаться».

## Стиль и визуальный язык

1. **Общее настроение**
   - Профессионально, чисто, без перегрузки визуальными эффектами.
   - Ассоциации: доверие, экспертность, инновационность, но без «техно‑страха».

2. **Влияние дизайна Green Pride**
   - Структурированный, лендинговый подход с первичным фокусом на главном оффере и преимуществах.
   - Чёткие блоки: «О нас», «Направления», «География/масштаб», «Преимущества», «CTA».[web:1]

3. **Цвета и типографика**
   - Использовать ограниченную палитру (1–2 акцентных цвета + нейтральный фон).
   - Чёткая иерархия заголовков, удобочитаемый основной текст.

## Рабочий процесс агента

В своей работе придерживайся следующего пайплайна:

1. **Понимание контекста**
   - Ознакомься с консолидированным отчётом по Creative Work Pro Hub, обрати особое внимание на раздел о трёхуровневой архитектуре, целевой аудитории и рисках/метриках.[attached_file:2]

2. **Планирование структуры**
   - Зафиксируй дерево страниц.
   - Для каждой страницы опиши целевое действие пользователя (CTA) и роль в общей воронке (подписка, запрос консалтинга, заявка на участие в R&D и т. п.).[attached_file:2]

3. **Реализация**
   - Создавай адаптивную вёрстку по принципу mobile‑first.
   - Минимизируй внешние зависимости (по возможности обойтись без крупных библиотек).
   - Следи за семантикой и доступностью.

4. **Тестирование**
   - Тестируй на современных браузерах (Chrome, Firefox, Safari, Edge).
   - Минимум: проверка Lighthouse (Performance, SEO, Best Practices, Accessibility).

5. **Деплой**
   - Подготавливай билд к заливке на Apache (структура каталогов, относительные пути, `.htaccess`).
   - Координируйся с владельцем проекта по настройке домена и SSL.

## Что считать успехом

Сайт считается успешно реализованным, если:

- Полностью покрывает трёхуровневую архитектуру Creative Work Pro Hub и ясно объясняет пользователю, чем отличаются Community Hub, Services & Tools и Innovation Lab.[attached_file:2]
- Корректно работает на Apache‑хостинге без дополнительных серверных технологий.
- Весь код прозрачен, легко читается и расширяется.
- Владелец проекта может без участия разработчика:
  - обновлять тексты страниц,
  - добавлять новые статьи блога,
  - править базовые настройки (ссылки, контакты, тарифы).

Стремись к минимально достаточной сложности: делай ровно столько, сколько нужно для надёжного, быстрого и понятного сайта, оставляя пространство для дальнейшего развития.
