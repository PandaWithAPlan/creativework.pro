# Notes

Рабочие заметки по мелким доработкам и идеям, которые можно реализовать после основных фаз (4–6). Это не обязательный backlog, а список возможных улучшений.

## Идеи по UX и визуалу

- Добавить отдельные визуальные стили для `.form-status--success` и `.form-status--error` в `css/styles.css` (цвет, иконки), чтобы статусы форм были заметнее.
- Немного смягчить анимацию появления секций (уменьшить смещение, duration) после того, как посмотрим живой билд на разных устройствах.
- Добавить лёгкий hover/active-эффект для карточек тарифов и pricing-toggle (переход фона/рамки), если не будет перегружать интерфейс.
- Для ссылок вида `link-underlined` рассмотреть вариант анимации подчёркивания при hover (но не во всех местах одновременно).

## Функциональные доработки форм

- Подключить реальный backend для форм (`api/contact.php` / внешний сервис), заменить текущую "фиктивную" отправку в `forms.js` на `fetch` с обработкой ошибок сети.
- Добавить базовый rate limiting/простую защиту от спама (honeypot-поле, простая капча) для контактных форм.
- Логировать события отправки форм и переключения тарифов в аналитику (GA4 или другое решение) через `analytics.js`.

## Pricing и навигация

- Добавить плавную анимацию при смене режима тарифа (monthly/yearly): переход цвета/размера цены с помощью CSS-транзишнов вместо мгновенной подмены текста.
- Рассмотреть отдельное микро-описание выгоды годовой подписки под переключателем, например "Экономия до 2 месяцев в годовом плане".
- Для навигации: при длинных страницах добавить "Back to top"-кнопку с тем же механизмом плавного скролла, если это потребуется по результатам тестов.
- Добавить skip-link (`<a class="skip-link" href="#page-home">Перейти к основному контенту</a>`) на всех ключевых страницах, чтобы пользователи клавиатуры могли быстро переходить к `main`, и оформить его в `styles.css` так, чтобы он был виден только при фокусе.

## Доступность и контент

- После финализации текстов прогнать страницы через автоматические проверки доступности (axe, Lighthouse) и собрать отдельный список правок.
- Пересмотреть alt-тексты партнёрских логотипов: возможно, где-то достаточно текстового списка вместо логотипов, чтобы не дублировать информацию.
- Рассмотреть отдельную короткую англоязычную версию лендинга (минимальный набор секций) для будущих тестов международной аудитории.

## Технический долг

- Уточнить структуру `js/` (при необходимости разделить на модули для навигации, форм, анимаций) и добавить короткие комментарии к публичным функциям.
- Добавить простые smoke-тесты (например, через Playwright) для проверки ключевых сценариев: скролл по якорям, переключатель тарифов, отправка форм.
